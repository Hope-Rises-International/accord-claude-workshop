Before We Start

One real task runs through the whole workshop. Each block adds a layer. By the end, you have a working Accord project workspace, an understanding of why Claude sometimes forgets things, and a personal skill that captures what worked today.

What you’ll build: A board-ready summary of Accord’s Principles of Practice — starting from nothing, then grounded in your real documents, then voiced in Accord’s tone. Along the way you’ll connect Claude to Gmail and Drive, learn how Claude’s memory actually works, and capture one workflow as a reusable skill.

What you need open: Two browser tabs. One for claude.ai (sign in to your usual Claude account). One for accordnetwork.org/about — we’ll be downloading two documents from there.

How this workshop runs

Six blocks. Most are 10–15 minutes. We don’t do theory first — concepts get introduced as they come up in the work. If something is unclear, raise it in the moment. If your screen does something different than mine, raise that too — Claude varies, and that’s part of the lesson.

The one idea behind today

You’re learning to direct work, not do it. Claude is a brilliant new hire with two PhDs who doesn’t know where the bathroom is. Today is about teaching it where the bathroom is — and once you do, it can do real work for Accord at a pace one person couldn’t.

The two documents we’ll use

  • Principles of Practice — Accord’s 11-principle operational framework. Download from accordnetwork.org/about (Principles section).
  • Accord GIK Standards (2019) — Seven Gifts-in-Kind standards. Same page.

Save both PDFs somewhere easy to find. You’ll upload them in Block 2.

1

Same Prompt, Two Models

The better model isn’t smarter — it’s more honest about what it doesn’t know.

We’re going to run the same prompt twice, on two different models, with no documents uploaded. Just a blank chat. The point is to see what Claude does with the same question before any Accord knowledge is added — and to feel the difference between a fast model and a thoughtful one.

Write a one-page briefing for a new board member on the Accord Network’s Principles of Practice. Cover what the document is, why it exists, the 11 principles, and how member organizations use it. Tone: professional, mission-aware, board-appropriate. One page maximum.
Step 1 — Sonnet 4.6

Open a new chat. Click the model selector and choose Sonnet 4.6. Paste the prompt. Read the output. It will be confident, well-structured, and at least partly invented — because Sonnet doesn’t have your Principles document in front of it. It’ll guess from what it knows about Christian relief networks in general.

Step 2 — Opus 4.7 Adaptive

Start a new chat. Select Opus 4.7 Adaptive — this is the current top-tier model. “Adaptive” means it automatically decides how hard to think based on the question: fast on easy ones, slow and careful on hard ones. Same prompt. Watch what happens — Opus may pause and search the web, find accordnetwork.org, pull real text, and cite it. You can click the thinking panel to watch its reasoning unfold.

What you should notice

✗ Sonnet, no context
Confident, plausible, partially wrong. Invents principle names, guesses founding year, generalizes. Reads like a generic faith-based nonprofit primer.
✓ Opus 4.7 Adaptive, no context
Pauses, searches, finds the real source, grounds the briefing in actual principle names with citations. Slower, more honest, more useful.

Factory knowledge — the new MacBook

Both models have absorbed millions of books, articles, and websites. That’s why each produced something plausible without any documents. But factory knowledge is generic — it knows about Christian relief networks, not about Accord specifically. That’s what we fix next.

When to use which

Sonnet 4.6 is your everyday workhorse — fast, capable, fine for most tasks once you’ve given it context. Opus 4.7 Adaptive is your heavy-lifter for research, complex analysis, and first drafts of important documents. It costs more (in tokens, in your plan’s limits), so save it for when quality matters most.

2

Context Changes Everything

A Sonnet with real documents beats an Opus with none. Context is the difference between guessing and knowing.

Now we give Claude your actual source material and re-run the same prompt. You’ll see the output transform — same model, same question, dramatically better result.

Step 1 — Create a project

In the left sidebar of Claude, click Projects → Create New Project. Name it “Accord Network.” A project is a persistent workspace — documents you add stay available across every chat inside it.

Step 2 — Upload the two documents

In the project, find the Files section. Drag Principles of Practice and Accord GIK Standards in (or click + and upload from device). Drop them into the project, not into a chat.

Step 3 — Re-run the prompt

Start a new chat inside the project. Stay on Sonnet 4.6 — we’re proving you don’t need Opus when you have real documents loaded. Paste the same board-member briefing prompt. Run it.

What you should see

The briefing now uses the actual principle names in the actual order. It quotes Accord’s own framing. It can reference specific sub-points (program staff care, indigenous empowerment, GIK valuation discipline) that it had no way to know in Block 1. The fabrication risk is gone because the source is in the room.

Context = the USB drive

Think of Claude as a new MacBook. Its training data is the factory software — broad and capable but generic. When you upload documents into a project, you’re plugging in a USB drive. While the chat is inside that project, Claude can read everything on the drive. Start a chat outside the project, the drive’s unplugged. That’s what projects are for: keeping the drive connected.

File format matters

Plain text and Word docs go in cleanly. PDFs work but lose accuracy — especially scanned ones. Excel works if structure is clean (single header row, no merged cells). For anything you want Claude to do math on, never send a PDF if you can send the source file. For documents and text, PDF is usually fine but DOCX is better.

Three Claude habits to know about

It won’t volunteer its full capability. If you tell it to write a one-page brief, it writes a one-page brief — even if it could also format it, send it, and file it. Ask for more, and you usually get more.

It accepts the limits you imply. If you frame a question narrowly, it answers narrowly. Stepping back and saying “what are we missing here?” often unlocks better thinking.

It will jump to writing before specifying. If the deliverable is important, ask Claude to draft a spec or outline first, then write. The five extra minutes saves rework.

3

A System Prompt: Set the Rules Once

Documents give Claude the facts. A system prompt gives Claude the rules — your tone, your framing, your guardrails. You set it once and it applies to every conversation in the project.

Right now every new chat in your Accord project starts the same way: helpful but generic. Claude doesn’t know how Accord talks, what to never say, who you are to your members. We fix that with a system prompt — instructions that load at the top of every conversation in the project.

Step 1 — Add the system prompt

In your Accord Network project, click the project settings (gear icon, or “Edit Project”). Find the Instructions field. Paste the prompt below. Save.

You are writing on behalf of the Accord Network, an association of 160+ Christ-centered relief and development organizations founded in 1977. We serve member organizations — we don’t direct them. Voice: professional, collaborative, rooted in shared evangelical conviction without being preachy. Confident about our role as a convener, humble about our members’ expertise on the ground. When summarizing or referencing our Principles of Practice or GIK Standards, draw from the uploaded documents — never from memory or general knowledge. Never fabricate statistics or attribute outcomes to specific member organizations unless those outcomes are in the source documents. Use “member organizations” rather than “clients,” “partners,” or “customers.” Use “the network” when referring to Accord collectively.
Step 2 — Re-run the prompt one more time

Start a new chat inside the same project. Paste the same board-member briefing prompt. Compare this output to the version from Block 2.

What changes

The tone tightens. The word “member organizations” appears where Claude might have said “partners” before. Claude is more careful about which claims it makes — because you told it not to fabricate. The voice starts to feel like Accord wrote it, not like a generic AI wrote about Accord.

The G-C-A-O formula

Most prompting frameworks boil down to four ingredients: Goal (what you want), Context (what Claude needs to know), Action (what role to play), Output (what format). Your system prompt handles the C and A permanently. Your day-to-day prompts only need the G and the O. You stop repeating yourself.

4

Connected Tools: Bring Claude to Your Work

You don’t move data to Claude. Claude comes to where your data already lives.

So far you’ve been uploading documents by hand. That works, but you’re the courier. Connectors eliminate that step — once Gmail and Google Drive are connected, Claude can search your inbox, read documents in your Drive, and create new files there directly.

Step 1 — Connect Gmail and Drive

Go to Settings → Connectors → Browse connectors. Click to connect Gmail and Google Drive. Each opens a Google authorization popup — approve the permissions. If a popup doesn’t appear, check for a “popup blocked” indicator in your browser’s address bar.

Step 2 — Verify

Start a new chat. Below the prompt bar, you should see icons for From Drive, From Calendar, and From Gmail. If they’re there, your connections are live.

Step 3 — Gmail in action

Think of a recent email thread with multiple participants — an onboarding discussion, a workshop planning thread, a member-services exchange. In your Accord project, paste:

Search my Gmail for the most recent thread about [your topic]. Summarize who was involved, what was discussed, and what action items are still outstanding. Then draft a brief follow-up email to the participants asking for status updates on the open items.

Claude searches your inbox, returns a summary, and drops a draft into your Gmail drafts folder. Check it before sending — but the heavy lifting is done.

Step 4 — Drive in action

Now ask Claude to find and use a document from your Drive:

Find the most recent version of our annual report or member-services overview in my Google Drive. Based on it, write a 200-word elevator pitch for a prospective member organization considering joining the network. Use our Principles of Practice as background for the tone.

Notice what just happened: Claude searched Drive, read the document, combined it with your uploaded Principles of Practice, and produced something new — all in one prompt.

One rule for everything Claude creates

Documents you create with Claude live inside the chat as artifacts. To make them real organizational knowledge, click the Drive icon on the artifact. It saves to your Drive. From there: if it’s a working document, convert to Google Doc (File → Save as Google Docs) and move to the right Shared Drive folder. If it’s a final deliverable, leave the format and move it. Either way, always move it. A document in My Drive disappears when you leave the org. A document in a Shared Drive becomes part of Accord.

5

The Context Window: Claude’s Working Memory

Every conversation lives inside a fixed-size window of working memory. Once you understand what fills it, you stop getting blindsided when Claude slows down or starts losing the thread.

So far we’ve been adding things to Claude’s awareness: documents, a system prompt, connected tools, a back-and-forth chat history. All of that lives in one place — the context window. It’s the amount of text Claude can actively hold in mind during a conversation. About 200,000 tokens, which sounds enormous, until you start filling it.

What’s actually in the window right now

SYSTEM FILES CHAT HISTORY TOOL CALLS FREE
Your system prompt Project files referenced Everything you and Claude have said Gmail/Drive results, search output Room left to think

The chat history is the part most people don’t feel growing. Every message you send and every response Claude gives gets carried forward into the next turn. After ten turns of back-and-forth, your “simple” chat is carrying ten turns of context every time it answers.

The accumulation trap

A long chat doesn’t fail dramatically — it degrades quietly. You’ll notice:

  • Responses get slower. Claude has more to chew on before answering.
  • Earlier details get fuzzy. Something you said in turn three may not survive turn fifteen.
  • Quality drifts. Claude starts repeating itself or contradicting earlier work.
  • You hit a wall. Eventually you see a “long conversation” warning or the chat refuses to continue.

Look at one of your chats from earlier in this workshop. The Block 4 chat probably already feels heavier than the Block 1 chat. That’s the accumulation trap in action.

Four escape routes

Route 1 — Start a fresh chat in the same project

This is the most common move. Your project files and system prompt stay in place; only the back-and-forth history drops. You lose the working memory of this conversation, but you keep the foundation. If the chat is getting sluggish, start fresh.

Route 2 — Summarize and paste forward

If the conversation has produced valuable thinking you don’t want to lose, end the chat by asking Claude:

Summarize this conversation as a handoff note for a fresh chat. Include: what we were trying to do, the key decisions we made, the documents and details Claude needs to pick up where we left off, and any open questions. Format it as a single block I can paste into a new chat.

Start a new chat, paste the summary as your first message, and continue. You’ve compressed twenty turns into one.

Route 3 — Lean on the project, not the chat

The whole reason projects exist is so you don’t have to carry knowledge in the chat itself. If something is going to be referenced repeatedly, put it in the Files section, not into the conversation. Files don’t fill the chat history — Claude reads them on demand when relevant. Your back-and-forth stays lean.

Route 4 — Use Opus 4.7 Adaptive for the heaviest lifts

Adaptive mode chooses how deeply to think based on the question. On a complex task with a heavy context window, it allocates more thinking — which uses the available room more efficiently than a faster, shallower model would. When you’ve got a lot loaded and you need a quality answer, Opus 4.7 Adaptive is the right tool.

The mental model

Think of the context window like a whiteboard. You can write on it, Claude can write on it, you can pin documents to it. But the whiteboard is a fixed size. Once it’s full, something has to come off before something new goes on. Long chats fill the whiteboard. Fresh chats start with a clean one. Projects keep the important pinned items in place across every clean whiteboard.

Practical guideline

If a chat has gone more than 15–20 substantive turns, or you notice responses slowing down, or Claude starts forgetting things you established earlier — that’s your cue. Summarize, start fresh, and keep moving. Don’t fight the window.

Capture a Skill Before You Close Today

Claude doesn’t carry memory between sessions. A great result today may look completely different tomorrow. That’s not a bug — it’s two equally talented people at the same desk making different choices. A skill writes down what worked so it happens the same way next time.

You’ve done five things today that worked. Pick one and turn it into a skill before you close the chat. Otherwise, the next time you try the same thing, you’ll get something different — and you’ll have to re-figure-out what to ask for.

Pick what to capture

Look back over today. Which of these gave you a result you’d want to repeat exactly?

  • The board-member briefing format from Block 3 — structured, voiced, grounded in your documents.
  • The Gmail summarize-and-draft workflow from Block 4 — thread → summary → follow-up draft, every time.
  • The Drive-find-and-elevator-pitch workflow from Block 4 — find latest doc, combine with Principles, produce a 200-word pitch.
  • The handoff summary pattern from Block 5 — capture a conversation as a paste-forward block for the next chat.
  • Something else you discovered along the way — if a particular phrasing or sequence got you a result you loved, that’s a skill waiting to be born.
Step 1 — Tell Claude what worked

Inside your Accord project, paste this and fill in the bracketed parts:

I just successfully [describe what you did — the workflow, the prompt, the steps, what made it work well]. I want to build a personal skill that captures this workflow so it works reliably every time. Write the skill for me — include a clear name, a description of when to use it, the exact instructions Claude should follow, and any phrasing or formatting rules that matter. Name it [your chosen name, e.g. “accord-member-briefing”].
Step 2 — Save it to your skills

When Claude finishes drafting the skill, look for a Copy to your skills option, or download the skill file and install it via Settings → Capabilities → Skills → Customize. The skill now lives in your Claude account. The next time you start a chat and ask for that kind of work, Claude reaches for the skill automatically. If it doesn’t, just say: “Use the [skill name] for this.”

Personal skill today, team skill tomorrow

The skill you just built is personal — only your Claude account uses it. That’s the right place to start. Try it for a week. If it holds up across multiple uses, bring it to Cassie. On a Teams plan, an admin can promote a personal skill to a team skill — meaning every Accord staffer’s Claude account picks it up automatically. That’s how good individual work becomes organizational standard.

Style vs. skill vs. custom instructions

Style is your voice — how Claude writes (concise, formal, etc.). Applies across every conversation, set in Claude’s settings.

Skill is your craft — how Claude structures a specific deliverable. What you just built.

Custom instructions are your personal preferences — how Claude should talk to you specifically. Lives in your own account settings.

Security note

Don’t download and install skill files from random websites. A skill is instructions Claude follows — an untrusted one could behave in unexpected ways. Build your own from your own work, or have Claude build them for you.

Troubleshooting

Connector toggled itself off? Open the + menu → Connectors and flip it back on. Known behavior, not your fault.

Claude asking questions instead of writing? Add “Don’t ask clarifying questions — write your best version with what you have” to the end of your prompt.

Files won’t upload? Drag them into the project’s Files section, not into a chat. Or use the + icon in Files.

Wrong document version returned from Drive? Be more specific in the prompt. Naming the year or version usually does it.

Hit a tool-call or token limit? Click Continue if you see the button. Otherwise start a fresh chat in the same project — your documents, system prompt, and skill persist.
